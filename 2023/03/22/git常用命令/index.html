git常用命令

git命令

本地操作

1.git init初始化本地仓库

  1. git add .添加到暂存区

  2. git status 查看git此时提交状态

  3. git commit -m 'zhushi' 提交到本地仓库

  4. git log 查看提交记录

  5. git reflog 操作记录

  6. 回退版本

    git reset --hard HEAD^ 回退到上一个版本

    git reset --hard HEAD~2 回退到上两个版本

    git reset --hard 6位版本号 回退到指定版本

  7. git remote add origin http://gitee.com/**** 添加一个远程的源，仅第一次创建的时候需要此命令

  8. git push 本地仓库同步到远程仓库

    注意区分：

    git push origin master 推送本地master分支到远程（origin）master分支上

    git push origin master:master作用等同于上一句，冒号前后分别代表本地分支和远程分支

    git push origin :master意思是推送本地空分支到远程master上，即等同于删除远程master分支。等同于git push origin --delete master

    附：

    git push -f f即force,强制推送，当版本不一致时，如果强制推送，本地分支会完全覆盖远程分支（慎用！！！）。正确做法是：先pull拉取远程分支，合并，再推送。

    git push origin --tags 添加标签

  9. git pull origin master 远程仓库拉取到本地仓库


！！注意：

git push origin master 与 git push -u origin master 的区别 ，即-u的含义

-u,即upstream,“上游”的意思，意思是指定本地分支与远程分支的关联关系，一旦设定以后，才能用git push等命令，反之，会报错。

-u完整写法git push --set-upstream origin master

！！！因此，在新建的初始化仓库中，首次推送要执行 git push -u origin master，非首次执行git push origin master


常见情景：

本人项目云端仓库备份代码

  1. git init初始化本地仓库

  2. git add .添加到暂存区

  3. git remote add origin http://gitee.com/**** 添加一个远程的源，仅第一次创建的时候需要此命令

  4. git commit -m "first commit"提交到本地仓库

  5. git push -u origin master（首次-u） 本地仓库push到远程仓库


协作共同完成的项目

  1. git clone http://****

  2. 正常 add commit push pull，无需初始化（默认克隆的远程仓库）


克隆别人的项目到本人远程仓库

  1. git clone http://****

  2. 本地进行修改

  3. 远程建立自己的仓库

  4. git add .添加到暂存区

  5. git remote rm origin删除本地仓库原有绑定的远程仓库

  6. git remote add origin http://gitee.com/**** 添加一个远程的源，仅第一次创建的时候需要此命令

  7. git push -u origin master（首次） 本地仓库push到远程仓库
